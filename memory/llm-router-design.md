# LLM Router 设计方案

## 目标
智能路由用户请求到最合适的 LLM，平衡成本、质量和延迟。

## 路由策略

### 1. 基于任务类型

| 任务类型 | 路由模型 | 理由 |
|---------|---------|------|
| 简单问答 | Gemma 3 4B (本地) | 快速、免费 |
| 代码生成 | DeepSeek-R1 7B (本地) | 推理能力强、免费 |
| 复杂推理 | DeepSeek-R1 7B (本地) | 深度思考、免费 |
| 长文本处理 | Kimi K2.5 (云端) | 256K 上下文 |
| 图像理解 | Kimi K2.5 (云端) | 支持多模态 |
| 高质量写作 | Kimi K2.5 (云端) | 输出质量高 |

### 2. 基于成本策略

- **经济模式**: 优先本地模型，云端作为 fallback
- **质量模式**: 优先云端模型，本地作为备选
- **自动模式**: 根据任务复杂度自动选择

### 3. 实现架构

```
用户请求 → Router → 任务分类 → 模型选择 → 执行 → 返回
                ↓
            成本/延迟/质量评估
```

## 技术方案

### 方案A: OpenClaw 内置 (推荐)
利用 OpenClaw 的 `models` 配置和别名系统，通过 prompt 路由。

### 方案B: 独立 Router 服务
搭建一个独立服务，封装多个 LLM API，对外提供统一接口。

### 方案C: MCP Server 实现
将 Router 实现为 MCP Server，Claude 通过 tools 调用。
