02 我们踩的三个坑（及解决方案）

坑 1：Memory 插件不可用 —— "昨天教的，今天忘"

症状：每次对话都像第一次见，完全记不住之前的上下文。

原因：Memory 插件未安装 + 配置中禁用了记忆搜索。

▲ Memory 插件修复前后对比

解决：

# 1. 安装 memory-core 插件 openclaw plugin install memory-core # 2. 启用记忆搜索 # 在 openclaw.json 中设置： "memorySearch": { "enabled": true }
效果：现在能记住你是谁、你们聊过什么、项目进展到哪了。

坑 2：飞书 Bindings 配置不安全 —— "谁都能@我干活"

症状：飞书群里任何人都能触发 AI，安全审计报 3 个 CRITICAL。

▲ 安全审计结果对比

解决：

"groupPolicy": "allowlist", "groupAllowlist": ["oc_xxx", "oc_yyy"], "groupAllowFrom": ["ou_你的用户ID"]
效果：安全审计从 3 个 CRITICAL 降到 0 个。

坑 3：没有备份 —— "更新后失忆怎么办"

症状：担心版本更新后，所有记忆和配置丢失。

解决：

  • GitHub 自动备份：每天 9:00 和 21:00 自动推送
  • 本地压缩备份：保留最近 10 个版本
  • 用户档案文件：单独记录关键信息
仓库地址：https://github.com/tangzheng202202/nailong-memory

05 核心感悟：养龙虾靠手艺，不是运气

"龙虾本身只是个壳，真正让它变强的，是你给它装的工具、喂的知识。"

我们的实践验证了这一点：

  1. 工具链要完整 —— 不能指望一个工具解决所有问题
  2. 记忆系统必须可靠 —— 否则永远是"第一次见"
  3. 备份是底线 —— 没有备份的配置都是耍流氓
  4. 安全不能妥协 —— 开放群聊 = 给别人开你的电脑

06 给新手的建议

▲ 定时任务工作流

如果你刚装 OpenClaw，按这个顺序：

第一步：基础配置（30分钟）

  • 配置模型（至少一个云端 + 一个本地）
  • 安装 Memory 插件
  • 设置安全策略（allowlist 不是可选）

第二步：工具链（1小时）

  • 安装浏览器自动化工具
  • 配置代理（访问国际源）
  • 测试常用功能（搜索、读取网页）

第三步：工作流（2小时）

  • 写一个你的专属 Skill
  • 设置定时任务
  • 配置备份（本地 + 云端）

第四步：持续迭代

  • 每次对话结束，让 AI 总结写入记忆
  • 每周Review一次工作流，优化提示词
  • 关注社区，替换不靠谱的 Skill
