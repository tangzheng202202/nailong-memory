## 02 真相一：80%的人卡在"安装"这一步

**数据说话：**

在播放量前20的视频中，**35%是安装教程**，且评论区最常见的问题是：
- "Docker启动失败怎么办？"
- "Windows能装吗？"
- "为什么一直连不上？"

**最讽刺的是：**

那个播放量最高（198.5万）的视频，标题叫《一个视频搞懂OpenClaw》，但评论区第一条热评是："看完还是不会装"。

这说明什么？

> **OpenClaw的门槛不在使用，而在安装。**

就像给你一台法拉利，但你连驾照都没有。

**我的建议：**

如果你技术背景不强，别折腾本地部署了。等云服务商出一键部署方案，或者找人帮你装好。

时间成本也是成本。

## 04 真相三：Memory插件是分水岭

**什么是Memory？**

简单说，就是让AI记住你们之前的对话。

没有Memory的OpenClaw，就像一条金鱼——每次对话都像是第一次见面。

**数据洞察：**

在50个视频中，**只有12个提到了Memory**，而这12个视频的**完播率明显高于其他**。

为什么？

因为装完Memory，OpenClaw才真正从"玩具"变成"工具"。

**一个真实的对比：**

| 场景 | 无Memory | 有Memory |
|------|---------|---------|
| 写周报 | 每次都要交代背景 | 直接说"继续写上周的周报" |
| 学编程 | 重复解释需求 | 记住你的水平和偏好 |
| 项目管理 | 项目进展丢失 | 自动跟踪任务状态 |

**但问题是：**

官方文档对Memory的配置说得不清不楚，很多人装完发现还是记不住。

我花了2个小时才搞定，过程如下：

```bash
# 1. 安装 memory-core 插件
openclaw plugin install memory-core

# 2. 修改配置
vim ~/.openclaw/openclaw.json
# 添加: "memorySearch": { "enabled": true }

# 3. 重启服务
openclaw gateway restart
```

**效果：**

现在我能记住用户是谁、聊过什么、项目进展到哪了。

## 05 那些UP主不会告诉你的坑

### 坑1：Token烧得飞快

有个UP主实测：10小时烧了200元Token。

如果你用GPT-4级别的模型，一次复杂对话可能就几块钱。

**解决方案：**
- 简单任务用本地模型（DeepSeek/Gemma）
- 复杂任务才上云端（Kimi/Claude）
- 设置预算上限，防止破产

### 坑2：Skill冲突

装多了Skill会冲突，报错信息还看不懂。

**解决方案：**
- 一次只装一个Skill，测试稳定后再加
- 定期清理不用的Skill
- 做好配置备份

### 坑3：安全风险

默认配置下，任何人@你的飞书机器人都能触发AI。

**解决方案：**
- 设置groupPolicy为allowlist
- 限制可访问的群组和用户
- 定期检查安全审计日志

## 07 给不同阶段人的建议

### 如果你还没装OpenClaw

**问自己三个问题：**
1. 我每天有多少重复性工作可以自动化？
2. 我愿意花多少时间学习配置？
3. 我的数据安全要求有多高？

如果答案都是"高"，再考虑安装。

### 如果你刚装好OpenClaw

**按这个顺序：**
1. 先装Memory插件（必须！）
2. 配置安全策略（防坑！）
3. 设置GitHub备份（防丢！）
4. 再装其他Skill

### 如果你已经用了一段时间

**考虑这些进阶玩法：**
- 多Agent协作（一个写代码，一个写文档）
- 定时任务自动化（早报、监控、报告）
- 自定义Skill（封装你的工作流）

- **你工作越有条理**，它帮你提效越明显

**一个月就是60小时。**
