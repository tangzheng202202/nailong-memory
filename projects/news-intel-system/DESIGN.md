# 新闻热点情报系统 - MVP设计

## 数据结构设计

### 热点事件表 (hot_events)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 事件ID | 自动编号 | 唯一标识 |
| 标题 | 文本 | 事件标题 |
| 来源平台 | 单选 | 微博/抖音/知乎/B站/百度 |
| 热度值 | 数字 | 平台原始热度 |
| 排名 | 数字 | 热搜排名 |
| 链接 | 链接 | 原始链接 |
| 抓取时间 | 日期时间 | 数据获取时间 |
| 分类标签 | 多选 | 娱乐/科技/社会/财经/体育 |
| 状态 | 单选 | 新发现/已查看/已采纳/已忽略 |
| 备注 | 文本 | 手动备注 |

### 选题决策表 (content_ideas)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 关联事件 | 关联字段 | 关联hot_events |
| 选题角度 | 文本 | 你的切入观点 |
| 适用平台 | 多选 | 抖音/公众号/视频号 |
| 预估效果 | 单选 | 高/中/低 |
| 执行状态 | 单选 | 待执行/制作中/已发布 |
| 发布链接 | 链接 | 发布后回填 |
| 实际效果 | 文本 | 播放量/点赞数等 |

## 抓取源

1. **微博热搜** - https://s.weibo.com/top/summary
2. **抖音热榜** - 需接口或爬虫
3. **知乎热榜** - https://www.zhihu.com/hot
4. **百度热搜** - https://top.baidu.com/
5. **B站热门** - https://www.bilibili.com/v/popular/all

## 推送策略

- **频率**: 每2小时抓取一次，每天 08:00/12:00/18:00 推送摘要
- **格式**: 飞书卡片，显示TOP10 + 变化趋势
- **交互**: 点击「查看详情」跳转多维表格

## 技术实现

- 爬虫: Python + requests/bs4/playwright
- 存储: 飞书多维表格 API
- 推送: 飞书机器人消息
- 调度: OpenClaw cron
